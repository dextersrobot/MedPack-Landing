What "Domain-Aware Queryable Compression" Means
ICD-10 Chapter Bitmask
A 32-bit integer stored per row group. Bit k = 1 iff the group contains any code starting with letter A+k. A chapter-E filter ORs bit 4 against the mask — O(1), no string comparison, no payload decompression. Parquet can only do generic string min/max, which is useless for ICD-10 (every large group spans A–Z).
NDC Labeler Bloom Filter
512-byte probabilistic filter (k=7 hashes, SHA-256) per row group, keyed on the 5-digit NDC labeler prefix. Zero false negatives. ~1% false positive rate. Allows skipping entire row groups that definitely do not contain a given manufacturer's drugs — capability unique to MedPack.
Column Pruning
Each column is stored as an independent chunk with its own byte offset. Reading 3 columns of 81 decompresses only 3/81 payloads — observed 16–23× speedup on wide files. Stats blocks (always uncompressed) are read without touching any payload.
Numeric & Date Range Skip
Min/max stored per column per row group, exactly as in Parquet. A range query skips any group whose [min, max] doesn't overlap the query range — no payload touched for skipped groups. Works for both financial amounts and ISO-8601 dates.
